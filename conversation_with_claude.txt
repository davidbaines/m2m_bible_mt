## Publishing

Successful models are shared on the Hugging Face Hub by a single gated command,
`samileides.publish` (run after a human review, not automatically). Decisions
agreed with David on 2026-07-06:

- **Quality gate**: publishing is refused unless every generated held-out book
  beats the source-copy baseline on chrF3. The command re-checks this from the
  run's `generated/metrics.csv`, so `samileides.generate` must have been run.
- **Licence gate**: publishing is refused unless every training translation
  carries a licence that permits sharing a *derived* model. The eBible
  `Redistributable` flag is `True` for all translations and only covers the
  source text, so the real signal is `licence_Licence_Type`. A trained model is
  treated as a derivative work. Shareable licences: `Public Domain`, `by`,
  `by-sa` (commercial use allowed); `by-nc` is shareable only non-commercially
  (opt-in with `--allow-nc`). Never shareable as a derivative: `by-nd`,
  `by-nc-nd`, `Unknown`, and anything else. Implication: a publishable run must
  train on a licence-filtered selection; runs that include, e.g., `by-nc-nd`
  German (`deuelbbk`) are research/comparison only. ShareAlike propagates, so
  any `by-sa` source forces the model licence to `cc-by-sa-4.0`; otherwise
  `cc-by-4.0` (any `by`) or `cc0-1.0` (all public domain). The model card lists
  every source translation and its licence.
- **Repository layout**: one repository per run under `DavidCBaines`, named
  `ebible_m2m-<experiment>` (e.g. `ebible_m2m-ie-base`). Public by default,
  since only redistributable data reaches a published model. The best models
  may later be mirrored to the `bible-nlp` organisation. Liedes' name is not
  used in any repository name.

  `MarianTokenizer.from_pretrained` work directly.

Why the source-copy scores are so low, and what they do and do not mean

The source-copy baseline takes the Koine Greek source verse and submits it unchanged as if it were the translation, then scores it against the target-language reference. chrF3 counts overlapping character n-grams. Greek is written in
the Greek script, English and German in Latin, Hindi in Devanagari, so character n-grams almost never he small non-zero values (0.25 to 0.34) are just incidental overlap: spaces, the occasional sharedpunctuation mark or Latin character. By construction this baseline sits on the floor.

The important caveat is that a low chrF does not mean the two texts share no meaning. chrF is a purely surface, orthographic measure. A perfect translation written in a different script would also score near zero. So the near-zero
baseline tells us only that copying the Greek is useless as Latin or Devanagari output, which is obvioy check that confirms the model is not degenerately echoing its input; it is not a demanding bar, andbeating it is not by itself impressive.

A much more informative baseline, which the spec already names but I have not yet implemented, is the best other training-language reference: take the same verse from the most orthographically similar training language (Dutch or
Danish for German, Gujarati or Marathi for Hindi) and score that against the reference. Because relatiry, that baseline is genuinely non-trivial, perhaps chrF3 in the twenties or thirties for closerelatives, and beating it is a real claim. I recommend I add it, and I can rescore the existing ie_base output with it so we have an honest floor. It is a small change.

On the absolute scores and SOTA

Two things are worth separating. First, the regime: chrF3 of 60 to 70 for general open-text systems is measured on high-resource, in-domain language pairs with abundant parallel data. Our task is generating an entire Old Testament for
a language the model saw none of, scored against a single specific translation. Even a fluent, correctse there are many valid ways to translate a verse, and human translations of the same verse comparedagainst each other often score in the 40 to 60 range rather than 100. So a single reference imposes a ceiling well below 100, and chrF3 around 40 is closer to a usable draft than the raw number suggests.

Second, the genuine headroom. This run is deliberately small: a 60M parameter model, trained from scratch for two hours, one Greek source per verse, only 34 languages. There is a lot of room to improve, which brings me to your SOTA
question.

Track A, the pure closed-text reproduction at scale (publishable as a clean result). In rough order of
